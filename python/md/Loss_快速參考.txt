═══════════════════════════════════════════════════════════
各個 Loss 的含義
═══════════════════════════════════════════════════════════

1. Loss/giou (0.51040) - 邊界框重疊損失
───────────────────────────────────────────────────────────
作用: 衡量預測框和真實框的重疊程度
關注: 整體框的形狀和位置（幾何關係）
優化: 讓預測框與真實框盡可能重疊
權重: 2.0

GIoU = IoU - |C - (預測框 ∪ 真實框)| / |C|
  • 改進版的 IoU，即使框不重疊也能提供梯度
  • 值越小越好（0 = 完全重疊）


2. Loss/l1 (0.06222) - 座標精確度損失
───────────────────────────────────────────────────────────
作用: 衡量預測座標和真實座標的絕對差異
關注: 每個座標的精確數值
優化: 精確調整邊界框的 4 個座標值
權重: 5.0 (最高！)

L1 = Σ|預測值 - 真實值| / 4
  • 關注具體座標值的精度
  • 提供細粒度的回歸約束
  • 與 GIoU 互補：GIoU 看整體，L1 看細節


3. Loss/location (0.95937) - 目標中心定位損失
───────────────────────────────────────────────────────────
作用: 衡量預測的目標中心熱圖與真實熱圖的差異
關注: 目標中心的粗略位置
優化: 快速找到目標在圖片中的大致區域
權重: 1.0

使用 Focal Loss:
  • 生成高斯熱圖（目標中心處值最高）
  • 預測熱圖與真實熱圖比較
  • 圖片中這個值最高 (0.95937) → 中心定位還需改進


4. pro_loss (0.10) - 提案選擇損失
───────────────────────────────────────────────────────────
作用: SGLATrack 特有，選擇最佳候選框
關注: 哪個候選框最可能是正確答案
優化: 提高模型的判別能力
權重: 0.2

邏輯:
  • 模型產生多個候選框
  • 找出相似度最高的候選框
  • 訓練提案權重集中在最佳候選框
  • 值較小 (0.10) → 模型已能較好選擇提案


5. Loss/total (2.31251) - 總損失
───────────────────────────────────────────────────────────
組成: 所有 Loss 的加權和

Loss/total = 2.0 × Loss/giou +      (1.02080)
             5.0 × Loss/l1 +         (0.31110)
             1.0 × Loss/location +   (0.95937)
             0.2 × pro_loss          (0.02000)
           = 2.31127 ≈ 2.31251 ✓

權重設計:
  • L1 最高 (5.0) → 精確座標很重要
  • GIoU 中等 (2.0) → 平衡整體重疊
  • Location 標準 (1.0) → 粗定位
  • pro 最小 (0.2) → 輔助作用

═══════════════════════════════════════════════════════════
Loss 之間的關係
═══════════════════════════════════════════════════════════

訓練流程（多階段定位）:

1. Location Loss → 快速找到目標區域（粗）
            ↓
2. GIoU Loss → 優化框的整體位置和大小（中）
            ↓
3. L1 Loss → 精細調整每個座標（細）
            ↓
4. pro_loss → 選擇最佳候選框（判別）

互補性:
┌─────────────┬──────────┬──────┬──────────────┐
│ Loss        │ 優化方向 │ 粒度 │ 主要作用     │
├─────────────┼──────────┼──────┼──────────────┤
│ Location    │ 目標中心 │ 粗   │ 定位目標區域 │
│ GIoU        │ 框重疊度 │ 中   │ 整體框質量   │
│ L1          │ 座標精度 │ 細   │ 精確座標     │
│ pro_loss    │ 提案選擇 │ -    │ 判別最佳候選 │
└─────────────┴──────────┴──────┴──────────────┘

為什麼需要這麼多 Loss？
═══════════════════════════════════════════════════════════
✓ 多階段定位：粗定位 → 整體框 → 精確座標
✓ 互補優化：每個 Loss 關注不同方面
✓ 提高魯棒性：避免陷入局部最優
✓ 追蹤任務複雜：需要同時處理定位、回歸、判別
