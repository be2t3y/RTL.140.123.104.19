# ❌ 訓練中斷錯誤分析與解決方案

## 🔍 錯誤原因

### Terminal 錯誤訊息
```
train.py: error: unrecognized arguments: --resume output/checkpoints/train/sglatrack/deit_distilled/sglatrack_ep0297.pth.tar
```

### 問題分析
**`train.py` 不接受 `--resume` 參數！**

這個專案的訓練腳本不支援命令列的 `--resume` 參數。

---

## ✅ 正確的 Finetune 方法

### 方法 1: 使用 `--script_prv` 和 `--config_prv`（推薦）

```bash
python tracking/train.py \
    --script sglatrack \
    --config experiments/sglatrack/uav123_coco_finetune.yaml \
    --save_dir output \
    --mode multiple \
    --nproc_per_node 1 \
    --script_prv sglatrack \
    --config_prv deit_distilled
```

**說明**:
- `--script_prv`: 之前的 script 名稱
- `--config_prv`: 之前的配置檔案名稱
- 系統會自動從 `output/checkpoints/train/sglatrack/deit_distilled/` 載入最新的權重

### 方法 2: 在配置檔案中指定

在 `uav123_coco_finetune.yaml` 中添加：

```yaml
TRAIN:
  PRETRAINED_PATH: "output/checkpoints/train/sglatrack/deit_distilled"
```

然後使用 `load_previous_ckpt=True` 參數訓練。

## 💡 為什麼之前的方法錯誤？

### 常見誤解

很多人（包括 GPT）會誤以為 PyTorch 訓練腳本都支援 `--resume` 參數，因為這是很多專案的慣例。

**但是**：
- 每個專案的實作不同
- 這個專案使用 `--script_prv` 和 `--config_prv` 的方式
- 需要查看實際的程式碼才能知道正確用法

### 圖片中的說明

圖片提到：
```
在 train 設定裡指定:
resume = sglatrack_ep0297.pth.tar
```

這是**概念性**的說明，實際上：
- 不是直接寫 `resume =`
- 而是使用 `PRETRAINED_PATH` 或 `--script_prv`/`--config_prv`
