# ❌ 測試命令錯誤修正

## 問題

您使用的命令格式不正確：
```bash
❌ 錯誤: python tracking/test.py --tracker_param sglatrack --dataset uav123 ...
```

## 正確格式

```bash
✅ 正確: python tracking/test.py <tracker_name> <tracker_param> --dataset_name <dataset> ...
```

---

## 🎯 正確的測試命令

### 1. 測試 UAV123 (使用現有模型)
```bash
conda activate sgla

python tracking/test.py sglatrack deit_distilled \
  --dataset_name uav123 --threads 4 --num_gpus 1
```

### 2. 測試 UAV123_10fps
```bash
python tracking/test.py sglatrack deit_distilled \
  --dataset_name uav123_10fps --threads 4 --num_gpus 1
```

### 3. 測試單個序列 (快速測試)
```bash
python tracking/test.py sglatrack deit_distilled \
  --dataset_name uav123 --sequence bike1 --threads 1 --num_gpus 1
```

---

## 📋 參數說明

### 必需參數 (位置參數)
- **tracker_name**: 追蹤器名稱，固定為 `sglatrack`
- **tracker_param**: 配置檔案名稱 (不含 .yaml 後綴)
  - `deit_distilled` - 使用原始配置和模型
  - `coco_only` - 使用 COCO 訓練的模型
  - `coco_uav123` - 使用 COCO+UAV123 訓練的模型

### 可選參數
- `--dataset_name`: 測試資料集名稱
  - `uav123` - UAV123 標準測試集 ✓
  - `uav123_10fps` - UAV123 10fps 版本 ✓
  - `uavdt` - UAVDT 資料集
  - `dtb70` - DTB70 資料集
  
- `--threads`: 平行執行緒數 (預設: 0，建議: 4-8)
- `--num_gpus`: 使用的 GPU 數量 (預設: 1)
- `--sequence`: 測試特定序列名稱
- `--debug`: 除錯等級 0-3

## 🔍 常見錯誤對照

| 錯誤命令 | 正確命令 |
|---------|---------|
| `--tracker_param sglatrack` | `sglatrack` (第一個位置參數) |
| `--dataset uav123` | `--dataset_name uav123` |
| 缺少配置名稱 | 需要第二個位置參數，如 `deit_distilled` |

---

## 💡 提示

1. **tracker_name** 和 **tracker_param** 必須是**前兩個參數**，不能用 `--` 開頭
2. 配置檔案名稱**不需要** `.yaml` 後綴
3. 資料集參數是 `--dataset_name`，不是 `--dataset`
4. 確保模型檔案存在於對應的 checkpoint 目錄

---

## 📁 模型檔案位置

根據您使用的配置，模型應該在：

```
output/checkpoints/train/sglatrack/<config_name>/sglatrack_ep<XXXX>.pth.tar
```

例如：
- `deit_distilled` → `output/checkpoints/train/sglatrack/deit_distilled/sglatrack_ep0297.pth.tar` ✓
- `coco_uav123` → `output/checkpoints/train/sglatrack/coco_uav123/sglatrack_ep0300.pth.tar`

TEST.EPOCH 在配置檔案中設定 (預設: 297 或 300)
