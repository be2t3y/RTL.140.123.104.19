## 🔢 30000 的計算依據

### 方法 1: 基於資料集大小比例

原始配置的比例：
```
大型資料集（LASOT + GOT10K）: 60000 samples
您的資料集（COCO + UAV123）: ?
```

**實際計算**：
```python
# COCO 圖片
COCO_images = 118,287

# UAV123 幀數（tracking 會取多幀組成 sample）
UAV123_frames = 81,727

# 合理的採樣數
# 通常設定為資料集大小的 20-50%
sample_per_epoch = (118287 + 81727) * 0.15 = ~30,000
```

### 方法 2: 基於訓練穩定性

**考量因素**：
1. **Batch size = 32**
2. **每 epoch 的 batch 數** = 30000 / 32 ≈ **937 batches**
3. **每 50 batches 顯示一次進度** → 每 epoch 顯示 18 次
4. **驗證間隔 = 5 epochs** → 每 5 epochs 驗證一次

**為什麼不是 20000？**
- 20000 / 32 ≈ 625 batches（如圖片中顯示的 `/625`）
- 對於 COCO（11 萬張圖）來說，**20000 可能太少**
- 模型每個 epoch 只能看到約 17% 的資料

**為什麼選 30000？**
- 30000 / 32 ≈ 937 batches
- 每個 epoch 能看到約 **25% 的 COCO 資料**
- 配合 100 epochs，總共能看到 25 遍完整資料
- **平衡訓練時間和模型收斂**
