## 📊 圖表內容

### 1. success_plot.pdf
- **橫軸**: Overlap threshold (0-1)
- **縱軸**: Overlap Precision [%]
- **曲線**: 不同重疊閾值下的成功率
- **數值**: AUC (66.90) 顯示在圖例中

### 2. precision_plot.pdf
- **橫軸**: Location error threshold [pixels] (0-50)
- **縱軸**: Distance Precision [%]
- **曲線**: 不同距離閾值下的精確度
- **數值**: Precision @ 20px (84.91) 顯示在圖例中

### 3. norm_precision_plot.pdf
- **橫軸**: Location error threshold (0-0.5, 歸一化)
- **縱軸**: Distance Precision [%]
- **曲線**: 歸一化距離閾值下的精確度
- **數值**: Norm Precision (82.56) 顯示在圖例中

## 🔧 我做的修正

### 問題
工作站沒有 LaTeX，原始代碼會報錯：
```
Failed to process string with tex because latex could not be found
```

### 解決方案
修改了 `lib/test/analysis/plot_results.py`：

1. **禁用 LaTeX 渲染**:
```python
plt.rcParams['text.usetex'] = False  # 原本是 True
```

2. **簡化標題格式**:
```python
title = plot_opts['title']  # 原本是 r"$\bf{%s}$" %(plot_opts['title'])
```

3. **移除 LaTeX 粗體**:
```python
# 註解掉 legend_text[-i] = r'\textbf{%s}'%(legend_text[-i])
```

### 結果
- ✅ 圖表成功生成
- ✅ 格式正常（只是字體不是 Times New Roman，改用 DejaVu Sans）
- ✅ 所有數值正確

## ⚠️ 注意事項

### 1. 字體警告（可忽略）
```
findfont: Font family ['Times New Roman'] not found. Falling back to DejaVu Sans.
```
- **不影響**: 圖表正常生成
- **原因**: 工作站沒有 Times New Roman 字體
- **替代**: 使用 DejaVu Sans（仍然清晰可讀）

### 2. 必須先測試
生成圖表前必須先執行測試：
```bash
python tracking/test.py sglatrack deit_distilled --dataset_name uav123 --threads 4 --num_gpus 1
```

### 3. 快取機制
- 第一次計算會生成快取: `eval_data.pkl`
- 之後會直接使用快取（很快）
- 要重新計算需刪除快取檔案
