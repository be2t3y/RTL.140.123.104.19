## 📋 模型還原選項

根據您的需求，有以下幾種方式還原模型：

### **選項 1: 下載已訓練好的模型權重（推薦用於測試）**

從論文作者提供的 Google Drive 下載預訓練模型：
- 📥 [模型下載連結](https://drive.google.com/drive/folders/1sHL7aFVZFwkPy6js48x-EKfoZC7oJc9X?usp=sharing)

下載後放置位置：
```bash
# 下載模型後放到以下路徑
output/checkpoints/train/sglatrack/deit_distilled/sglatrack_ep0297.pth.tar
```

可用模型版本：
- **SGLATrack-DeiT*** (推薦) - UAV123 AUC: 66.9%
- **SGLATrack-ViT** - UAV123 AUC: 66.1%
- **SGLATrack-EVA** - UAV123 AUC: 65.1%

### **選項 2: 從頭訓練模型**

如果您想要重新訓練模型，需要：

#### 步驟 1: 準備訓練資料集
下載並整理以下資料集到 `data/` 目錄：
```
data/
├── lasot/          # LaSOT 資料集
├── got10k/         # GOT-10k 資料集
│   ├── train/
│   ├── val/
│   └── test/
├── coco/           # COCO 資料集
│   ├── annotations/
│   └── images/
└── trackingnet/    # TrackingNet 資料集
    ├── TRAIN_0/
    ├── TRAIN_1/
    └── ...
```

#### 步驟 2: 執行訓練
```bash
# 啟用環境
conda activate sgla

# 單 GPU 訓練
python tracking/train.py \
  --script sglatrack \
  --config deit_distilled \
  --save_dir ./output \
  --mode single \
  --use_wandb 0

# 多 GPU 訓練（4 GPU）
python tracking/train.py \
  --script sglatrack \
  --config deit_distilled \
  --save_dir ./output \
  --mode multiple \
  --nproc_per_node 4 \
  --use_wandb 0
```

訓練參數（來自 `deit_distilled.yaml`）：
- **Batch Size**: 32
- **訓練輪數**: 300 epochs
- **學習率**: 0.0004
- **優化器**: AdamW
- **每輪樣本數**: 60,000

## 🔧 疑難排解

### 問題 1: 找不到模型檔案
**解決方案**: 
- 確認已下載模型權重並放到正確路徑
- 或者檢查訓練是否完成並產生了 checkpoint

### 問題 2: CUDA 記憶體不足
**解決方案**:
- 減少 batch size（編輯 yaml 檔案中的 `TRAIN.BATCH_SIZE`）
- 減少 GPU 數量或使用更小的模型

### 問題 3: 資料集路徑錯誤
**解決方案**:
- 檢查 `lib/test/evaluation/local.py` 中的路徑設定
- 確保資料集已下載並解壓到正確位置
