# 訓練路徑錯誤解決指南

## 📋 錯誤總結

### 錯誤 1: 配置檔案路徑重複 ❌

```
ValueError: /home/junjie/01_Research/SGLATrack-main/experiments/sglatrack/experiments/sglatrack/uav123_coco_finetune.yaml.yaml doesn't exist.
```

**問題分析**：
1. **路徑重複**: `experiments/sglatrack/experiments/sglatrack/`
2. **後綴重複**: `uav123_coco_finetune.yaml.yaml`

### 錯誤 2: 按 Enter 無法等待 ⏭️

```bash
read -p "按 Enter 開始訓練..."
# 直接繼續執行，沒有等待使用者輸入
```

**問題分析**：
- `read -p` 在非互動式 shell 腳本中不會正確等待
- 需要使用倒數計時或其他方式給使用者時間

---

## 🔧 解決方案

### 1. 配置檔案路徑修正

**原因**：`train.py` 會自動添加 `experiments/{script}/{config}.yaml` 路徑

```python
# lib/train/run_training.py 第 62 行
settings.cfg_file = os.path.join(prj_dir, 'experiments/%s/%s.yaml' % (script_name, config_name))
```

**正確用法**：
```bash
# ❌ 錯誤：提供完整路徑
CONFIG="experiments/sglatrack/uav123_coco_finetune.yaml"

# ✓ 正確：只提供配置名稱（不含路徑和 .yaml 後綴）
CONFIG="uav123_coco_finetune"
```

### 2. 等待提示修正

**解決方案**：使用倒數計時代替 `read`

```bash
# ❌ 舊版：read 不會等待
read -p "按 Enter 開始訓練..."

# ✓ 新版：倒數計時
echo "按 Ctrl+C 取消，或等待 5 秒後自動開始訓練..."
for i in 5 4 3 2 1; do
    echo -n "$i... "
    sleep 1
done
echo ""
echo "開始訓練..."
```

## 📝 修正後的訓練腳本

### train_uav123_finetune.sh

```bash
#!/bin/bash
# UAV123 + COCO Finetune 訓練腳本

set -e

# 配置（✓ 只需要配置檔案名稱）
CONFIG="uav123_coco_finetune"  # 不含路徑和 .yaml
NUM_GPUS=1

# 倒數計時
echo "按 Ctrl+C 取消，或等待 5 秒後自動開始訓練..."
for i in 5 4 3 2 1; do
    echo -n "$i... "
    sleep 1
done
echo ""
echo "開始訓練..."

# 執行訓練
python tracking/train.py \
    --script sglatrack \
    --config "$CONFIG" \
    --save_dir output \
    --mode multiple \
    --nproc_per_node $NUM_GPUS \
    --script_prv sglatrack \
    --config_prv deit_distilled
```

## 🎓 重點整理

### train.py 的參數規則

1. **--config 參數**：
   - ✓ 只需配置名稱：`uav123_coco_finetune`
   - ❌ 不要完整路徑：`experiments/sglatrack/uav123_coco_finetune.yaml`
   - ❌ 不要 .yaml 後綴

2. **程式會自動組合**：
   ```python
   # 自動組合成：
   experiments/{script_name}/{config_name}.yaml
   
   # 範例：
   experiments/sglatrack/uav123_coco_finetune.yaml
   ```

3. **為什麼會重複**：
   ```bash
   # 錯誤的 CONFIG
   CONFIG="experiments/sglatrack/uav123_coco_finetune.yaml"
   
   # 程式處理後變成
   experiments/sglatrack/experiments/sglatrack/uav123_coco_finetune.yaml.yaml
   #                     ^^^^^^^^^^^^^^^^^^^^^^^^^ 這部分重複了
   #                                                                   ^^^^^ 後綴也重複了
   ```

### Shell 腳本最佳實踐

1. **倒數計時** 比 `read -p` 更可靠
2. **提供取消選項** (`Ctrl+C`)
3. **顯示清楚的進度訊息**
4. **檢查返回代碼** 判斷訓練成功或失敗
