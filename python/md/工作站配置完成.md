## 🚀 訓練指令

### 方案 A: 只用 COCO 訓練 (推薦)

#### 單 GPU 訓練
```bash
conda activate sgla

python tracking/train.py \
  --script sglatrack \
  --config coco_only \
  --save_dir ./output \
  --mode single \
  --use_wandb 0
```

#### 多 GPU 訓練 (4 張 GPU)
```bash
conda activate sgla

python tracking/train.py \
  --script sglatrack \
  --config coco_only \
  --save_dir ./output \
  --mode multiple \
  --nproc_per_node 4 \
  --use_wandb 0
```

#### 調整 GPU 數量
如果您有不同數量的 GPU，修改 `--nproc_per_node` 參數：
- 2 張 GPU: `--nproc_per_node 2`
- 8 張 GPU: `--nproc_per_node 8`

---

### 方案 B: 先測試訓練流程 (強烈建議)

在開始完整訓練前，建議先測試 5 個 epoch：

#### 步驟 1: 建立測試配置
```bash
cp experiments/sglatrack/coco_only.yaml experiments/sglatrack/coco_test.yaml
```

#### 步驟 2: 修改測試配置
編輯 `experiments/sglatrack/coco_test.yaml`，修改以下參數：
```yaml
TRAIN:
  EPOCH: 5              # 改為 5
  PRINT_INTERVAL: 10    # 改為 10（更頻繁輸出）
  VAL_EPOCH_INTERVAL: 2 # 改為 2（每 2 個 epoch 驗證）
```

#### 步驟 3: 執行測試訓練
```bash
conda activate sgla

python tracking/train.py \
  --script sglatrack \
  --config coco_test \
  --save_dir ./output \
  --mode single \
  --use_wandb 0
```

**測試通過後再開始完整訓練！**

## ⚙️ 重要參數調整

### 如果 GPU 記憶體不足

編輯 `experiments/sglatrack/coco_only.yaml`:

```yaml
TRAIN:
  BATCH_SIZE: 16    # 原本 32，改為 16
  # 或更小
  BATCH_SIZE: 8     # 如果還不夠
```

**記憶體需求參考**:
- Batch size 32: ~11GB GPU 記憶體
- Batch size 16: ~6GB GPU 記憶體
- Batch size 8: ~3GB GPU 記憶體

### 調整訓練輪數

如果想快速測試：
```yaml
TRAIN:
  EPOCH: 100        # 原本 300，可改為 100
```

### 調整驗證頻率

```yaml
TRAIN:
  VAL_EPOCH_INTERVAL: 10  # 每 10 個 epoch 驗證一次
```

## ⚠️ 常見問題

### Q1: 訓練過程中斷了怎麼辦？
**A**: 訓練會自動從最後一個 checkpoint 恢復，只需重新執行訓練命令。

### Q2: 如何只保留部分 checkpoint？
**A**: 修改配置檔案（需要查看訓練腳本的具體實作）或手動刪除舊的：
```bash
# 只保留最後 10 個
cd output/checkpoints/train/sglatrack/coco_only/
ls -t *.pth.tar | tail -n +11 | xargs rm
```

### Q3: 可以同時使用 COCO 和其他資料集嗎？
**A**: 可以！修改 `experiments/sglatrack/coco_only.yaml`:
```yaml
TRAIN:
  DATASETS_NAME:
  - COCO17
  # - GOT10K_vottrain  # 如果有 GOT10k
  # - LASOT             # 如果有 LaSOT
  DATASETS_RATIO:
  - 1
  # - 1
  # - 1
```

### Q4: 訓練時 GPU 使用率低怎麼辦？
**A**: 
1. 增加 `NUM_WORKER` (資料載入執行緒)
2. 增加 `BATCH_SIZE`
3. 使用 SSD 存放資料集

### Q5: 記憶體洩漏或記憶體使用增長？
**A**: 
1. 減少 `BATCH_SIZE`
2. 減少 `NUM_WORKER`
3. 確認 Python 版本和 PyTorch 版本相容
