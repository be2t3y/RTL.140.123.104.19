# 配置文件錯誤修正 - PRETRAINED_PATH

## ❌ 錯誤訊息

```
ValueError: PRETRAINED_PATH not exist in config.py
```

## 🔍 錯誤原因

在 `experiments/sglatrack/uav123_coco_finetune.yaml` 中使用了不支援的參數：

```yaml
TRAIN:
  PRETRAINED_PATH: "output/checkpoints/train/sglatrack/deit_distilled"  # ❌ 錯誤
```

**為什麼錯誤**：
- `lib/config/sglatrack/config.py` 中沒有定義 `PRETRAINED_PATH` 參數
- 配置文件中的所有參數都必須在 `config.py` 中預先定義
- 當載入 YAML 時，會檢查每個參數是否存在於 `config.py` 中

## ✅ 解決方案

### 方案：使用 `--script_prv` 和 `--config_prv` 參數

**正確做法**：
```bash
python tracking/train.py \
    --script sglatrack \
    --config "uav123_coco_finetune" \
    --save_dir output \
    --mode multiple \
    --nproc_per_node 1 \
    --script_prv sglatrack \      # ✓ 指定要載入的前一個訓練的 script
    --config_prv deit_distilled   # ✓ 指定要載入的前一個訓練的 config
```

**這樣會載入**：
```
output/checkpoints/train/sglatrack/deit_distilled/sglatrack_ep0297.pth.tar
```

## 💡 重點整理

1. **配置參數限制**：
   - YAML 配置文件中只能使用 `config.py` 中定義的參數
   - 自定義參數會導致 `ValueError: XXX not exist in config.py`

2. **載入預訓練權重的正確方式**：
   - ✓ 使用 `--script_prv` 和 `--config_prv` 命令列參數
   - ❌ 不要在 YAML 中使用 `PRETRAINED_PATH`

3. **腳本修改**：
   - ✓ 移除倒數計時，馬上開始訓練
   - ✓ 更流暢的使用者體驗
