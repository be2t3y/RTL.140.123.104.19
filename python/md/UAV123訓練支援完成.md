## 🚀 訓練指令

### 方案 A: COCO + UAV123 訓練 (推薦)

這個配置結合了通用物體追蹤 (COCO) 和 UAV 特定場景 (UAV123)：

#### 單 GPU 訓練
```bash
conda activate sgla

python tracking/train.py \
  --script sglatrack \
  --config coco_uav123 \
  --save_dir ./output \
  --mode single \
  --use_wandb 0
```

#### 多 GPU 訓練
```bash
# 4 張 GPU
python tracking/train.py \
  --script sglatrack \
  --config coco_uav123 \
  --save_dir ./output \
  --mode multiple \
  --nproc_per_node 4 \
  --use_wandb 0

# 2 張 GPU
python tracking/train.py \
  --script sglatrack \
  --config coco_uav123 \
  --save_dir ./output \
  --mode multiple \
  --nproc_per_node 2 \
  --use_wandb 0
```

---

### 方案 B: 只用 UAV123 訓練

如果您想只用 UAV123 進行訓練（快速實驗）：

#### 建立配置檔案
```bash
cp experiments/sglatrack/coco_uav123.yaml experiments/sglatrack/uav123_only.yaml
```

#### 編輯 `experiments/sglatrack/uav123_only.yaml`
```yaml
  TRAIN:
    DATASETS_NAME:
    - UAV123
    DATASETS_RATIO:
    - 1
    SAMPLE_PER_EPOCH: 30000  # 減少樣本數
```

#### 執行訓練
```bash
python tracking/train.py \
  --script sglatrack \
  --config uav123_only \
  --save_dir ./output \
  --mode single \
  --use_wandb 0
```

## 📈 訓練資料配比

### COCO + UAV123 配置
```yaml
TRAIN:
  DATASETS_NAME:
  - COCO17        # 118,287 訓練圖片
  - UAV123        # 91 視訊序列
  DATASETS_RATIO:
  - 1             # COCO 權重
  - 1             # UAV123 權重
  SAMPLE_PER_EPOCH: 60000
```

### 調整資料集比例

如果想增加 UAV123 的權重：

```yaml
DATASETS_RATIO:
- 1             # COCO
- 2             # UAV123 (2倍權重)
```

如果想減少 UAV123 的權重：

```yaml
DATASETS_RATIO:
- 2             # COCO (2倍權重)
- 1             # UAV123
```

### 使用 COCO + UAV123 訓練的優勢

## ⚠️ 注意事項

### 1. UAV123 資料量較小
- 91 個序列相對於 COCO (118k 圖片) 較小
- 建議與 COCO 組合使用以獲得更好的泛化能力

### 2. 視訊 vs 圖片
- UAV123 是視訊序列
- COCO 是靜態圖片
- 訓練時會從視訊中取樣幀對

### 3. 訓練時間
使用 COCO + UAV123 (4 GPU):
- 測試訓練 (5 epochs): ~2-3 小時
- 完整訓練 (300 epochs): ~7-10 天

### 4. 磁碟空間
- Checkpoints: ~75 GB (300 epochs)
- 確保有足夠空間
