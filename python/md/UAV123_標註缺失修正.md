## ❌ 問題現象


## 🔍 錯誤原因

### 問題 1: 資料夾和標註檔案不匹配

**實際情況**：
```bash
# data_seq 中的序列（124 個）
data/uav123/UAV123/data_seq/UAV123/
├── person7/          ✓ 有這個資料夾
├── person7_1/
├── person7_2/
└── ...

# anno 中的標註檔案（只有一部分）
data/uav123/UAV123/anno/UAV123/
├── person7_1.txt     ✓ 有這個標註
├── person7_2.txt     ✓ 有這個標註
└── person7.txt       ❌ 沒有這個標註！
```

**為什麼會錯誤**：
1. `_build_sequence_list` 函數列出 `data_seq/UAV123/` 中的**所有**資料夾
2. 包括了 `person7`、`person8` 等沒有對應標註檔案的序列
3. 訓練時嘗試讀取 `person7.txt`，但檔案不存在
4. 程式崩潰，無法開始訓練

---

## ✅ 解決方案

### 修正 `_build_sequence_list` 函數

**檔案**: `lib/train/dataset/uav123.py` (第 41-51 行)

```python
def _build_sequence_list(self):
    """Build list of sequences from UAV123 dataset."""
    seq_dir = os.path.join(self.root, 'data_seq', 'UAV123')
    anno_dir = os.path.join(self.root, 'anno', 'UAV123')
    
    if not os.path.exists(seq_dir):
        raise RuntimeError(f'UAV123 sequence directory not found: {seq_dir}')
    
    if not os.path.exists(anno_dir):
        raise RuntimeError(f'UAV123 annotation directory not found: {anno_dir}')
    
    # ✓ 只包含有對應標註檔案的序列
    all_sequences = sorted([d for d in os.listdir(seq_dir) 
                           if os.path.isdir(os.path.join(seq_dir, d))])
    
    sequence_list = [seq for seq in all_sequences 
                    if os.path.exists(os.path.join(anno_dir, f'{seq}.txt'))]
    
    print(f"UAV123: Found {len(sequence_list)}/{len(all_sequences)} sequences with annotations")
    
    return sequence_list
```

**關鍵修改**：
1. 檢查 `anno_dir` 是否存在
2. **過濾序列清單**：只包含有對應 `.txt` 標註檔案的序列
3. 列印訊息顯示找到多少有效序列

## 💡 關於 "No matching checkpoint file found"

您可能注意到圖片中也有這個訊息。這是**正常的提示訊息**，表示：

1. **使用 `--script_prv` 和 `--config_prv` 時**：
   - 程式會嘗試在新的訓練目錄中尋找 checkpoint
   - 找不到是正常的（因為是第一次訓練這個配置）
   - **但會成功載入預訓練權重**（從 `deit_distilled` 目錄）

2. **不影響 finetune**：
   - 這只是說在 `output/checkpoints/train/sglatrack/uav123_coco_finetune/` 中沒有找到 checkpoint
   - 預訓練權重仍然會從 `deit_distilled` 目錄載入
   - **這是預期行為，不是錯誤**

3. **如何確認有載入預訓練權重**：
   - 檢查訓練開始前的模型參數列表（您的 terminal 第 775-870 行）
   - 這些參數來自 `deit_distilled` 的預訓練模型
   - ✓ 表示成功載入了預訓練權重

## 🎓 重點整理

### 為什麼沒顯示訓練進度

| 原因 | 說明 |
|------|------|
| **序列和標註不匹配** | `data_seq` 有 124 個序列，但只有 108 個有標註 |
| **程式沒有過濾** | 原始程式碼假設所有序列都有標註 |
| **訓練失敗** | 載入 `person7` 時找不到 `person7.txt`，程式崩潰 |
| **沒有 batch 輸出** | 訓練在第一個 batch 前就失敗了 |

### 修正後的行為

1. ✓ 自動過濾沒有標註的序列
2. ✓ 只使用 108 個有完整標註的序列
3. ✓ 訓練正常開始
4. ✓ 顯示每 50 個 batch 的訓練進度
5. ✓ 包含 FPS、Loss、IoU 等指標
